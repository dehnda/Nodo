Universal Parameters Reference

All Nodo nodes inherit certain universal parameters that appear at the top of their property panels.

Parameter Hierarchy
1. Universal Group (all nodes)
2. Special Parameters (Component, Group Type, Primitive Type - if applicable)
3. Mode Selector (if node has modes)
4. Node-Specific Parameters (organized by category)

1. Universal "Group" Parameter

Every node inherits this from SOPNode constructor. It appears at the very top.

2. Parameter Categories

üîµ Universal "Group" Parameter
Backend: register_parameter(define_string_parameter("group", ""))
Appears on: ALL 44 nodes (inherited from SOPNode constructor)
UI Position: Always at top, in highlighted "Universal" section
Purpose: Filter which elements the operation affects
Empty value: Operate on all elements of the node's class
üü¢ "Component" Parameter
Backend: register_parameter(define_int_parameter("component", 0).options({...}))
Appears on: ~10 nodes (Wrangle, Attribute Create/Delete, Color, etc.)
UI Position: In Universal section, below Group
Purpose: Which geometry component the operation executes on
Options: Points, Primitives, Edges, Vertices
üü° "Group Type" Parameter
Backend: register_parameter(define_int_parameter("group_type", 0).options({...}))
Appears on: ~10 nodes (all Group nodes, Split, etc.)
UI Position: In Universal section, below Group
Purpose: Disambiguate which type of group we're referencing/creating
Options: Primitives, Points, Edges
üü£ "Primitive Type" Parameter
Backend: register_parameter(define_int_parameter("primitive_type", 0).options({...}))
Appears on: ~6 generator nodes (Sphere, Cube, Grid, etc.)
UI Position: In Universal section, below Group
Purpose: Control output geometry representation
Options: Polygons, Points, Edges
